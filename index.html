Nomenclatura para Variables y Constantes

Guía práctica para nombrar variables y constantes de manera clara y consistente en diferentes lenguajes de programación.

Importancia de una nomenclatura adecuada

Una buena nomenclatura mejora la legibilidad del código, facilita el mantenimiento y promueve la colaboración en equipo. A continuación, se presentan las mejores prácticas para nombrar variables y constantes.

Python

  • Variables: Usa snake_case (minúsculas con guiones bajos). Ejemplo: user_name
  • Constantes: Usa MAYÚSCULAS con guiones bajos. Ejemplo: MAX_USERS
  • Nombres descriptivos y específicos.
  • Evita nombres genéricos como temp o data.
user_age = 25
MAX_LOGIN_ATTEMPTS = 3

JavaScript

  • Variables: Usa camelCase. Ejemplo: userProfile
  • Constantes: Usa MAYÚSCULAS con guiones bajos. Ejemplo: API_KEY
  • Prefiere nombres que indiquen propósito.
  • Usa const para constantes inmutables.
let userProfile = {};
const API_KEY = "xyz123";

Java

  • Variables: Usa camelCase. Ejemplo: employeeId
  • Constantes: Usa MAYÚSCULAS con guiones bajos. Ejemplo: MAX_CONNECTIONS
  • Usa nombres claros y concisos.
  • Evita abreviaturas ambiguas.
int employeeId = 1001;
static final int MAX_CONNECTIONS = 50;

Mejores prácticas generales

  • Usa nombres que reflejen la intención y el propósito.
  • Mantén consistencia en el estilo de nomenclatura dentro del proyecto.
  • Evita nombres demasiado largos o redundantes.
  • Revisa las guías de estilo específicas del lenguaje (PEP 8, Airbnb, Oracle).
