Nomenclatura para Clases, Métodos y Comentarios

Guía para nombrar clases, métodos y estructurar comentarios de manera clara y profesional en diferentes lenguajes de programación.

¿Por qué es importante la nomenclatura?

Una nomenclatura clara para clases, métodos y comentarios mejora la legibilidad, facilita el mantenimiento y asegura un código comprensible para todos los desarrolladores.

Python

  • Clases: Usa PascalCase. Ejemplo: UserProfile
  • Métodos: Usa snake_case, verbos para acciones. Ejemplo: calculate_total
  • Comentarios: Usa docstrings para clases/métodos, comentarios en línea para aclaraciones breves.

Java

  • Clases: Usa PascalCase. Ejemplo: EmployeeManager
  • Métodos: Usa camelCase, verbos para acciones. Ejemplo: getEmployeeId
  • Comentarios: Usa JavaDoc para clases/métodos, comentarios en línea para claridad.
/** Clase para gestionar empleados */
public class EmployeeManager {

    // Retorna el ID
    return 0;

Mejores prácticas generales

  • Clases: Usa nombres sustantivos que describan la entidad (ej. OrderProcessor).
  • Métodos: Usa verbos que indiquen la acción (ej. processOrder).
  • Comentarios: Sé conciso, explica el "por qué" más que el "qué", usa herramientas como JavaDoc o JSDoc.
  • Mantén consistencia en el estilo dentro del proyecto.
